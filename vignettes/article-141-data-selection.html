Topics covered: - Subsetting rows and columns - Removing rows with incomplete data - Ordering rows ### Scenario Suppose we are interested in the `course` data table to investigate the question: If a student were ever enrolled in Engineering at Institution B, what was their level (first-year, second-year, etc.) when passing one of the core math courses (Calculus 1, 2, 3 and Ordinary Differential Equations)? Always a good idea to examine the basic data first. We see that the data frame is also a data.table object (midfieldr default) with 3.3M observations and 12 variables---all character class except `hours_course`. ## Getting started Load the `course` data set from the midfielddata package. ```{r} #| cache: true library(midfielddata) data(course) str(course) ``` I use the data set in three formats: as a base R data.frame object `course`, as a data.table object `course_dtblbl` (as-is), and as a tibble object `course_dplyr`. ```{r class.source="base", class.output="base"} course_base <- data.frame(course) ``` ```{r} #| collapse: true course_base <- data.frame(course) course_dtbl <- copy(course) course_dplyr <- dplyr::tibble(course) class(course_base) class(course_dtbl) class(course_dplyr) ``` ## Subsetting rows and columns ### Solution 1: base R To be able to use all base R semantics, we start by converting the `course` dataset from its default class "data.table" to the more basic class "data.frame". The base R solution works by using the `[,]` operator. It is a good idea to always include the `drop = FALSE` argument to ensure the result is a data frame. (Otherwise, when selecting a single column, the default behavior is to return a vector.) When typing the `columns_we_want` values, we can also reorder the columns. Here, I relocated the discipline column.
