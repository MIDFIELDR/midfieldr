Introduction

Instructional programs are encoded in cip, a midfieldr data frame of character variables listing the 2, 4, and 6-digit CIP codes and program names in the MIDFIELD database. To learn more, open the help page by running ?cip.

Taking a glimpse() of cip reveals that we have 1544 unique observations (6-digit programs) and shows the first few values of the 6 variables.

glimpse(cip)
#> Observations: 1,544
#> Variables: 6
#> $ CIP2     <chr> "01", "01", "01", "01", "01", "01", "01", "01", "01",...
#> $ CIP2name <chr> "Agriculture/Agricultural Operations and Related Scie...
#> $ CIP4     <chr> "0100", "0101", "0101", "0101", "0101", "0101", "0101...
#> $ CIP4name <chr> "Agriculture/General", "Agricultural Business and Man...
#> $ CIP6     <chr> "010000", "010101", "010102", "010103", "010104", "01...
#> $ CIP6name <chr> "Agriculture - General", "Agricultural Business and M...

cip_filter() filters cip data frame for the programs we wish to study. The function has two arguments:

  • series Search terms. A character vector of codes or keywords.
  • data A searchable data frame (cip is the default).

cip_filter() searches all columns in data for the strings in series and returns a data frame whose rows contain the search terms. Filtering is performed by dplyr::filter() with pattern detection by stringr::str_detect().

With no arguments, cip_filter() returns the unique 2-digit series and the first entry of the corresponding 4- and 6-digit series.

cip_filter()
#> Error in arrange(., CIP6): could not find function "arrange"

Search by keywords

Because cip_filter() searches all columns in data, we can search by keyword to find the CIP codes that match the programs under study. The search is case-sensitive.

For example, if we wanted all Civil Engineering programs, we might try keywords “civil” and “Civil”.

# series with a keyword
df <- cip_filter(series = c("Civil", "civil"))
#> Error in arrange(., CIP2, CIP4, CIP6): could not find function "arrange"
kable(df)
#> Error in seq_len(m): argument must be coercible to non-negative integer
#> Error in seq_len(m): argument must be coercible to non-negative integer

The search returns some programs with Civilization in their names as well as Engineering Technology. If we wanted Civil Engineering only, we should search for CIPs that start with the 4-digit “1408” code. We use the regular expression notation ^ to match the start of the strings.

# series with a 4-digit code
df <- cip_filter(series = "^1408")
#> Error in arrange(., CIP2, CIP4, CIP6): could not find function "arrange"
kable(df)
#> Error in seq_len(m): argument must be coercible to non-negative integer
#> Error in seq_len(m): argument must be coercible to non-negative integer

In the next example, suppose we want to study programs relating to German culture, history, politics, language, and literature. Using “German” for the series value yields

# series with a keyword
df <- cip_filter(series = c("German", "german"))
#> Error in arrange(., CIP2, CIP4, CIP6): could not find function "arrange"
kable(df)
#> Error in seq_len(m): argument must be coercible to non-negative integer
#> Error in seq_len(m): argument must be coercible to non-negative integer

From the result we find the specific 6-digit codes we want and refine the search to obtain,

# series with a 6-digit code
df <- cip_filter(series = c("050125", "160501", "160599"))
#> Error in arrange(., CIP2, CIP4, CIP6): could not find function "arrange"
kable(df)
#> Error in seq_len(m): argument must be coercible to non-negative integer
#> Error in seq_len(m): argument must be coercible to non-negative integer

Search by CIP codes

As illustrated above, assigning CIP codes to series yields a data frame of all rows from data that contain those character strings. Invalid codes—and valid codes not in the midfieldr cip dataset—are silently ignored.

Specifying 6-digit codes yields a data frame with a row for each 6-digit code.

# 6-digit example
df <- cip_filter(series = c("140802", "140803"))
#> Error in arrange(., CIP2, CIP4, CIP6): could not find function "arrange"
kable(df)
#> Error in seq_len(m): argument must be coercible to non-negative integer
#> Error in seq_len(m): argument must be coercible to non-negative integer

Specifying 4-digit codes yields a data frame with as many rows as there are rows that include the search strings.

# 4-digit example
df <- cip_filter(series = c("1407", "1408"))
#> Error in arrange(., CIP2, CIP4, CIP6): could not find function "arrange"
kable(df)
#> Error in seq_len(m): argument must be coercible to non-negative integer
#> Error in seq_len(m): argument must be coercible to non-negative integer

This search, in addition to the desired Engineering programs, returned two Foreign Language programs because their 6-digit codes contain “1407” and “1408”. We use the regular expression notation ^ to match the start of the strings.

# 4-digit example matching the start of the string
df <- cip_filter(series = c("^1407", "^1408"))
#> Error in arrange(., CIP2, CIP4, CIP6): could not find function "arrange"
kable(df)
#> Error in seq_len(m): argument must be coercible to non-negative integer
#> Error in seq_len(m): argument must be coercible to non-negative integer

The 2-digit series represent the most general groupings of related programs. Here, the result includes all History codes and programs.

# 2-digit example
df <- cip_filter("^54")
#> Error in arrange(., CIP2, CIP4, CIP6): could not find function "arrange"
kable(df)
#> Error in seq_len(m): argument must be coercible to non-negative integer
#> Error in seq_len(m): argument must be coercible to non-negative integer

The series argument can include any combination of 2, 4, and 6-digit codes.

# a series with 2, 4, and 6-digits specified
df <- cip_filter(c("^24", "^4102", "^450202"))
#> Error in arrange(., CIP2, CIP4, CIP6): could not find function "arrange"
kable(df)
#> Error in seq_len(m): argument must be coercible to non-negative integer
#> Error in seq_len(m): argument must be coercible to non-negative integer

A series assigned numerically is coerced to characters.

# series as a sequence
df <- cip_filter(series = seq(540102, 540110, 1))
#> Error in arrange(., CIP2, CIP4, CIP6): could not find function "arrange"
kable(df)
#> Error in seq_len(m): argument must be coercible to non-negative integer
#> Error in seq_len(m): argument must be coercible to non-negative integer

Invalid codes in the sequence, e.g., 540109 and 540110, are silently ignored.

Using a previous result as a data argument

A data frame produced by cip_filter() can be used as the data argument in a subsequent use of cip_filter().

df1 <- cip_filter("^49")
#> Error in arrange(., CIP2, CIP4, CIP6): could not find function "arrange"

Suppose, after viewing the full “49” series above we decide we only need the “4903” subset. We can use the data frame df1 from above as the data argument.

# use a previous result as a data argument
df2 <- cip_filter(series = "4903", data = df1)
#> Error in cip_filter(series = "4903", data = df1): object 'df1' not found
kable(df2)
#> Error in kable(df2): object 'df2' not found
#> Error in kable(df2, "html"): object 'df2' not found
